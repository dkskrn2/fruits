# 온라인 과일 커머스 제안 문서 제작 실행계획

## 1. 문서 목적
- 과일 온라인 커머스 시장을 다각도로 분석하고, 실행 가능한 **3가지 전략 방향성**을 도출한다.
- 최종 목표는 **온브릭스를 뛰어넘는 경쟁 전략**을 설계하는 것이다.
- 단순 시장 개요가 아니라 아래를 포함한다.
  - 주요 온라인 커머스별 특징/포지션/장단점
  - 시장 키워드 및 소비자 관심 과일
  - 네이버 스마트스토어 인기 카테고리/상품 특성

## 2. 최종 산출물
1. 시장 분석 본문 문서 (근거 데이터 포함)
2. 경쟁사/포지션 비교표
3. 스마트스토어 인기 상품 분석표
4. 전략 방향성 3안 (각 안별 실행안 포함)
5. 추천 우선안 1개 + 90일 실행 로드맵

### 산출물 형식 고정
- 모든 단계 완료 후 최종 결과 문서는 **Markdown(`.md`) 파일**로 작성한다.
- 권장 파일명: `final_online_fruit_commerce_proposal.md`
- 중간 산출물도 가능하면 `.md`로 통일한다.
- 에이전트 운영 기준 문서는 `fruit_proposal_agent_setup.md`를 따른다.
- 최종 산출물 구성 고정:
  - `final_online_fruit_commerce_proposal.md` (본 보고서)
  - `appendix_research_log.md` (검색/수집 로그)
  - `appendix_competitor_benchmark.md` (온브릭스 포함 경쟁사 비교)
  - `appendix_smartstore_catalog.md` (카테고리/상품 분석 원본)

## 3. 분석 범위 (Scope)

### 3-1. 커머스 분석
- 대상: 온브릭스(핵심 벤치마크) 포함 **핵심 경쟁사 10개** 심층 분석
- 항목:
  - 상품 구조(가성비/프리미엄/구독/선물)
  - 가격대, 중량, 원산지, 당도/품종 표기 방식
  - 배송 정책(새벽/당일/묶음배송/무료배송 조건)
  - 상세페이지 특징, 리뷰 운영, 재구매 유도 장치
  - 포지션, 핵심 전략, 강점/약점, 실행 난이도, 온브릭스 대비 우위/열위

### 3-2. 수요/관심도 분석
- 네이버 데이터랩/쇼핑 인사이트 기반 검색량 및 관심도 추이
- 계절별 인기 과일 및 이벤트성 수요(명절/선물/건강 키워드)
- 리뷰 텍스트에서 반복되는 구매 결정 키워드 추출

### 3-3. 스마트스토어 실전 분석
- 인기 카테고리 Top 제품군 추출
- 베스트 상품의 공통 포맷 분석:
  - 썸네일/타이틀 키워드
  - 가격 정책(정가/할인/쿠폰/묶음)
  - 신뢰 장치(산지 표기, 당도 보증, 교환/환불 정책)

## 4. 데이터 소스
- 공공/연구: KREI, aT, 통계청, 농식품부
- 플랫폼: 네이버 데이터랩, 네이버 쇼핑/스마트스토어
- 경쟁사: 주요 커머스 공개 페이지/앱
- 내부 문서: `research/`, `analysis/`, `strategy/`, `input/*.pdf`
- 사용자 탐색 레퍼런스: `input/0308/resorce.md` (참고용 아이디어/관찰 메모)

### 데이터 참고 우선순위
1. **현재 폴더 내부 문서**를 1차 근거로 우선 분석한다.
   - 특히 현재 상황(기존 전략/가정/표현 방식)의 기준선은 `output/business_plan.html`을 우선 참고한다.
   - 범용 사업계획 구조가 필요할 때만 `business_proposal_plan.md`를 보조 참고한다.
2. 내부 문서에서 비어 있는 항목만 외부 데이터로 보완한다.
3. 최종 문서에는 내부 문서 근거와 외부 근거를 구분해 표기한다.

### 4-1. 외부 탐색 방식 (Google/Naver 키워드 확장)
- 원칙: 한 주제를 단일 검색어로 끝내지 않고, **관점별 키워드 묶음**으로 확장 탐색한다.
- 기본 관점:
  - 소비자 관점: 인기 과일, 후기, 선물, 구독, 재구매
  - 판매자 관점: 스마트스토어 운영, 상세페이지, 리뷰 전략, 광고 효율
  - 운영 관점: 산지, 당도, 포장, 배송 클레임, 콜드체인
  - 경쟁 관점: 커머스별 포지션, 가격대, USP, 멤버십
  - 데이터 관점: 통계/리포트/정책 자료
- 채널별 탐색:
  - Google: 시장 리포트/뉴스/해외 사례 탐색
  - Naver: 쇼핑(상품/카테고리), 블로그/카페(실사용 맥락), 데이터랩(관심도)

### 4-2. 키워드 운용 규칙
- Seed 키워드 예시:
  - `과일 온라인`, `스마트스토어 과일`, `과일 구독`, `프리미엄 과일 선물`, `당도 보증`, `산지직송`, `과일 도매 온라인`
- 확장 패턴:
  - `[과일명] + 후기/추천/선물/당도/산지/가격`
  - `[플랫폼명] + 과일 + 인기/베스트/카테고리`
  - `[타겟] + 과일 + 니즈` (예: 1인가구, 맞벌이, 시니어)
- 수집 로그를 남긴다:
  - `검색일시 / 채널(Google|Naver) / 키워드 / 핵심 인사이트 / 출처 링크 / 전략 시사점`
- `input/0308/resorce.md`는 참고 샘플로 사용하되, 동일한 형식으로 탐색 결과를 누적 기록한다.

### 4-3. 수집 데이터 스키마 (필수 컬럼)
- 검색/자료 로그:
  - `수집일`, `채널`, `키워드`, `핵심 사실`, `정량 수치`, `출처 URL`, `신뢰도(A/B/C)`, `전략 시사점`
- 경쟁사 벤치마크:
  - `브랜드`, `포지션`, `핵심전략`, `대표 카테고리`, `가격대`, `대표 SKU 3개`, `배송 정책`, `차별점`, `약점`, `성장장치(구독/멤버십/CRM)`, `온브릭스 대비`
- 스마트스토어 상품:
  - `카테고리`, `상품명`, `가격`, `용량/중량`, `원산지`, `당도/품종 표기`, `리뷰수`, `평점`, `배송조건`, `후기 키워드`

### 4-4. 데이터 정제 규칙
- 중복 규칙: 동일 URL 또는 동일 상품명+옵션 조합은 1건으로 집계한다.
- 최신성 규칙: 날짜 확인 가능한 자료를 우선 사용하며, 구버전 데이터는 별도 표시한다.
- 출처 규칙: 최종 본문에 들어가는 수치는 최소 2개 근거 또는 1차 출처로 검증한다.

### 4-5. 노이즈 제거 규칙 (깊이 확보용)
- 동일 주장 반복 기사/블로그는 1건만 채택하고, 반복 건은 참고 목록으로만 보관한다.
- 광고성/협찬성 콘텐츠는 `신뢰도 C`로 분리하고 핵심 근거에서 제외한다.
- 숫자 없는 의견성 문장은 전략 근거로 단독 사용하지 않는다.
- 키워드 로그 건수 채우기 목적의 표면적 수집을 금지한다.

## 5. 분석 프레임

### 5-1. 포지셔닝 맵
- 축 A: 가격(가성비 ↔ 프리미엄)
- 축 B: 상품구성(범용 ↔ 과일 전문)
- 축 C(보조): 운영강점(물류속도 ↔ 큐레이션/신뢰)

### 5-2. 경쟁력 평가 항목
- 상품력
- 가격 경쟁력
- 배송/풀필먼트
- 신뢰 장치(품질보증/리뷰/브랜드)
- 재구매 시스템(구독/멤버십/CRM)
- 운영 난이도 및 수익성
- **온브릭스 대비 우위 여부** (기능/가격/브랜딩/운영)

### 5-3. 전략안 평가 기준(점수화)
- 시장성 (30)
- 차별성 (20)
- 실행 가능성 (20)
- 수익성 (20)
- 리스크 관리 용이성 (10)

### 5-4. 온브릭스 초과 달성 판단 기준
- 같은 고객군에서 온브릭스 대비 더 강한 가치제안이 있는가
- 온브릭스 대비 전환율/재구매율을 높일 운영 장치가 있는가
- 온브릭스 대비 가격 또는 품질에서 명확한 선택 이유가 있는가
- 온브릭스가 대응하기 어려운 구조적 해자(공급망/데이터/브랜드)가 있는가

### 5-5. 온브릭스 비교 KPI 프레임
- 비교 KPI:
  - `객단가(AOV)`, `재구매율(30/60일)`, `리뷰 평점`, `배송 리드타임`, `클레임률`, `구독 전환율`
- 산출 원칙:
  - 가능한 KPI는 온브릭스 `기준선(Baseline)`과 `우리 목표(Target)`를 함께 기재한다.
  - 정량 수치가 없으면 정성 비교로 임시 표기하되, 후속 검증 TODO를 남긴다.

### 5-6. 경쟁사 심층도 루브릭 (10개 공통)
- 각 경쟁사별 최소 작성 단위:
  - 포지션 1개, 핵심전략 3개, 가격대 구간 3개, 대표 SKU 3개
  - 강점 3개, 약점 3개, 온브릭스 대비 우위/열위 3개
  - 근거 출처 8개 이상(1차/2차 혼합)
- 심층도 점수(100점):
  - 포지션 정확도(20), 전략 해석력(20), 가격/상품 근거성(20), 운영/배송 분석(20), 온브릭스 비교 명확성(20)
- 통과 기준:
  - 평균 75점 이상 + 개별 경쟁사 65점 미만 없음

## 6. 전략 방향성 3안 도출 구조

### 전략안 A: 프리미엄 큐레이션형
- 핵심가치: 품질/당도/선물 경험
- 타겟: 프리미엄 소비자, 선물 수요층
- 수익원: 고마진 세트, 시즌 기획전

### 전략안 B: 가성비 대량/정기배송형
- 핵심가치: 가격+편의성
- 타겟: 맞벌이/1인가구/반복 구매층
- 수익원: 정기구독, 묶음판매, 재구매 자동화

### 전략안 C: B2B2C 하이브리드형
- 핵심가치: 기존 도매 인프라를 디지털 전환
- 타겟: 식당/소매 파트너 + 일반 소비자
- 수익원: B2B 안정 매출 + B2C 확장 매출

## 7. 실행 일정 (딥 리서치 기본안: 4주)
1. **Week 1**: 스코프 고정 + 내부 문서/PDF 전수 분석 + 가설 트리 작성
2. **Week 2**: Google/Naver 키워드 심층 탐색 + **핵심 경쟁사 10개 심층 벤치마크**
3. **Week 3**: 스마트스토어 카테고리/상품 정밀 분석 + 리뷰 키워드/가격/패키징 패턴화
4. **Week 4**: 전략 3안 설계 + 온브릭스 초과 달성 논리 검증 + 최종 `md` 문서화

### 7-1. 단계별 탐색 볼륨 목표 (딥 리서치)

|------|------------------|------------------|

| **Week 2 외부 키워드 탐색** | Google/Naver 합산 검색 로그 120건 | Google/Naver 합산 검색 로그 180건 |
| **Week 2 경쟁사 벤치마크(심층)** | 온브릭스 포함 8개 심층 분석 | 온브릭스 포함 **10개 심층 분석** |

### 7-2. 깊이 우선 원칙 (Breadth < Depth)
- 숫자 목표보다 **분석 밀도**를 우선한다.
- 경쟁사 분석은 `많이 보기`보다 `핵심 10개 깊게 보기`를 기본 원칙으로 한다.
- 목표 미달이라도 심층도가 높으면 통과 가능, 목표 초과라도 피상적이면 미통과로 본다.

### 7-3. 단기안(참고)
- 빠른 검증이 필요할 때만 1주 축약 플랜을 사용한다.
- 본 프로젝트 기본 모드는 `딥 리서치(4주)`로 고정한다.

### 7-4. 단계 종료 품질 게이트
- Week 1 종료 조건:
  - 핵심 팩트 100개 이상, 인용 가능한 근거 표기 완료
- Week 2 종료 조건:
  - 검색 로그 120건 이상
  - 온브릭스 포함 핵심 경쟁사 10개 중 최소 8개 심층 프로파일 완료
  - 각 경쟁사별 `포지션/핵심전략/가격대/대표 SKU 3개/배송정책/강약점/온브릭스 대비` 기재 완료
- Week 3 종료 조건:
  - 스마트스토어 상품 160개 이상 분석
  - 카테고리별 가격대/후기 키워드/상품구성 패턴 요약 완료
- Week 4 종료 조건:
