# 과일 섭취 환경 설계 프로젝트 에이전트 운영 설정

## 1. 목적
- 본 작업의 질문은 `어떻게 과일을 팔까`가 아니라 `어떻게 사람들이 과일에 돈을 내고 먹는 환경을 만들까`이다.
- 따라서 핵심 산출물은 상품 판매안이 아니라 `세그먼트별 섭취 환경 설계안`, `마찰 제거 장치`, `반복 섭취 루틴 설계`다.
- 본 문서는 이 목표를 실행하기 위한 에이전트 역할, 입력, 출력, 종료 조건을 정의한다.

## 2. 운영 원칙
- 세분화 기준은 연령보다 `먹는 상황`, `지불 명분`, `섭취 마찰`, `반복 가능성`을 우선한다.
- 모든 전략은 `왜 지금 과일에 돈을 써야 하는가`를 설명할 수 있어야 한다.
- 각 세그먼트는 반드시 다음 5가지를 갖는다.
  - 핵심 상황
  - 지불을 막는 마찰
  - 돈을 내게 만드는 명분
  - 환경 설계 장치
  - 검증 방식
- 아이디어는 많게 내기보다 `실제로 루틴을 바꾸는 환경` 중심으로 좁힌다.

## 3. 핵심 질문
- 과일이 왜 `건강하지만 애매한 식품`으로 남아 있는가
- 어떤 상황에서는 사람들이 과일에 기꺼이 돈을 내는가
- 어떤 세그먼트는 `즉시 공략`, 어떤 세그먼트는 `후속 실험`으로 나눠야 하는가
- 구매 이전, 섭취 순간, 재구매 루프 중 어디를 바꾸면 가장 효과가 큰가

## 4. 에이전트 구성

### Agent 0. Orchestrator
- 역할: 전체 범위 관리, 세그먼트 우선순위 조정, 최종 산출물 통합
- 입력:
  - 본 문서
  - `fruit_consumption_environment_plan.md`
  - 각 에이전트 중간 산출물
- 출력:
  - 최종 구조안
  - 우선 실행 세그먼트 목록
  - 미해결 리스크 목록
- 종료 조건:
  - 세그먼트별 전략이 중복 없이 정리됨
  - 즉시 실행군과 후속 실험군이 분리됨

### Agent 1. Segment Discovery Agent
- 역할: 사람을 `과일을 먹는 맥락` 기준으로 세분화
- 입력:
  - `appendix_consumer_voice.md`
  - `final_online_fruit_commerce_proposal.md`
  - 내부 리서치 및 기존 타겟 문서
- 출력:
  - 세그먼트 후보 리스트
  - 세그먼트 병합/분리 기준
- 핵심 작업:
  - 연령/성별 중심 세분화를 상황 중심 세분화로 재구성
  - `선물`, `가족 건강`, `아이 간식`, `자기관리`, `직장 간식`, `고령 가족 돌봄` 같은 섭취 맥락 분리
  - 세그먼트별 구매 빈도와 루틴화 가능성 추정
- 종료 조건:
  - 우선 검토할 세그먼트 5~7개 정의

### Agent 2. Barrier Mapping Agent
- 역할: 각 세그먼트가 과일에 돈을 안 쓰는 이유를 구조적으로 정리
- 입력:
  - Agent 1 산출물
  - 소비자 보이스
  - 기존 제안서의 페인포인트 정리
- 출력:
  - 세그먼트별 마찰 지도
- 핵심 작업:
  - 가격 부담
  - 손질/보관 불편
  - 맛 편차 불안
  - 섭취 루틴 부재
  - 대체재 대비 명분 부족
  - 가족 내 의사결정 복잡성
- 종료 조건:
  - 세그먼트별 핵심 마찰 3개 이내로 압축

### Agent 3. Environment Design Agent
- 역할: 사람을 먹게 만드는 환경 장치를 세그먼트별로 설계
- 입력:
  - Agent 1, 2 산출물
- 출력:
  - 세그먼트별 환경 설계안
- 핵심 작업:
  - 소포장/번들/용도별 구성
  - 보관/후숙/손질 보조
  - 시간대별 섭취 제안
  - 가족 공유 구조
  - 구매 이후 섭취 리마인드
  - 실패 방지 장치
- 종료 조건:
  - 세그먼트별 `먹는 환경 장치` 최소 3개 도출

### Agent 4. Offer and Pricing Agent
- 역할: 환경 설계를 실제 지불 제안으로 번역
- 입력:
  - Agent 3 산출물
- 출력:
  - 세그먼트별 오퍼 구조
- 핵심 작업:
  - 어떤 형태로 팔아야 지불 저항이 낮아지는지 설계
  - SKU, 묶음, 가격대, 진입 상품, 반복 상품 분리
  - `비싸지만 먹을 만함`이 아니라 `돈 내도 합리적`인 구조 설계
- 종료 조건:
  - 세그먼트별 진입 오퍼와 반복 오퍼 도출

### Agent 5. Habit Loop Agent
- 역할: 1회 구매를 반복 섭취 루프로 연결
- 입력:
  - Agent 3, 4 산출물
- 출력:
  - 세그먼트별 루틴화 시나리오
- 핵심 작업:
  - 첫 구매 계기
  - 첫 섭취 성공 장치
  - 재구매 트리거
  - 알림/리마인드/추천 흐름
  - 가족 내 공유/소비 분배 메커니즘
- 종료 조건:
  - 각 세그먼트별 `첫 구매 -> 첫 섭취 -> 반복 섭취` 흐름 완성

### Agent 6. Channel and Messaging Agent
- 역할: 세그먼트별 진입 채널과 메시지 구조 설계
- 입력:
  - Agent 1~5 산출물
- 출력:
  - 세그먼트별 메시지 맵
  - 채널 우선순위
- 핵심 작업:
  - 어떤 세그먼트는 상세페이지, 어떤 세그먼트는 콘텐츠, 어떤 세그먼트는 리마인드가 핵심인지 구분
  - `건강`만으로는 약한 세그먼트와 `안심`이 먹히는 세그먼트 분리
- 종료 조건:
  - 세그먼트별 핵심 문장 1개와 채널 1~2개 확정

### Agent 7. Validation Agent
- 역할: 세그먼트 전략의 실행 가능성 검증
- 입력:
  - 전체 산출물
- 출력:
  - 검증 메모
  - 우선 실험 과제
- 핵심 작업:
  - 너무 추상적인 환경 설계 제거
  - 실제 운영 불가능한 전제 제거
  - 세그먼트 과잉 분화 점검
- 종료 조건:
  - 즉시 실험 가능한 과제만 남김

## 5. 추천 세그먼트 초안
- `아침 대체형`: 바쁜 출근/등교 전 빠르게 먹을 수 있어야 돈을 낸다.
- `아이 간식 관리형`: 과자 대체와 부모의 죄책감 완화가 핵심 명분이다.
- `가족 건강 루틴형`: 장보기 안에서 자연스럽게 반복 소비가 일어나야 한다.
- `1인 자기관리형`: 건강, 체중관리, 가벼운 섭취 명분이 중요하다.
- `직장/오피스 간식형`: 휴대성, 나눔, 간편성이 핵심이다.
- `돌봄/안부형`: 고령 부모, 회복기 가족 등 돌봄 상황에서 지불 명분이 강해진다.
- `선물-전환형`: 선물 구매를 일상 섭취 루프로 연결하는 브리지 역할이다.

## 6. 작업 순서
1. Agent 0이 범위와 산출물 형식을 고정한다.
2. Agent 1이 세그먼트 후보를 만든다.
3. Agent 2가 세그먼트별 마찰 지도를 만든다.
4. Agent 3이 먹는 환경 장치를 설계한다.
5. Agent 4가 오퍼와 가격 구조를 붙인다.
6. Agent 5가 반복 섭취 루프를 설계한다.
7. Agent 6이 채널과 메시지를 정리한다.
8. Agent 7이 실행 가능성을 검토한다.
9. Agent 0이 최종 우선순위를 확정한다.

## 7. 최종 산출물
- 세그먼트 맵
- 세그먼트별 환경 설계표
- 세그먼트별 메시지/오퍼/루틴 설계안
- 우선 실험 과제 3~5개
- 즉시 실행군 / 후속 실험군 구분표
