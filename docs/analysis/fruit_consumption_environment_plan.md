# 과일 섭취 환경 설계 실행 계획

## 1. 문제 재정의
- 기존 질문: `어떻게 과일을 더 팔까`
- 새 질문: `어떻게 사람들이 과일에 돈을 내고 먹는 환경을 만들까`

이 질문은 판매보다 더 앞단의 문제를 다룬다. 사람들은 과일의 필요성을 몰라서 안 먹는 것이 아니라, 과일이 `비싸고`, `귀찮고`, `실패 가능성이 있고`, `대체재 대비 명분이 약한 상태`로 놓여 있기 때문에 돈을 내지 않는다. 따라서 해결은 광고보다 `먹는 환경 설계`에 있다.

## 2. 목표
- 사람들을 섭취 맥락 중심으로 세분화한다.
- 각 집단이 과일에 돈을 내지 않는 구조적 이유를 정리한다.
- 각 집단별로 `지불 명분`, `환경 장치`, `반복 루틴`, `진입 상품`을 설계한다.
- 최종적으로 `즉시 공략 가능한 세그먼트`와 `후속 실험 세그먼트`를 구분한다.

## 3. 핵심 가설

### 가설 1. 과일은 취향 문제가 아니라 환경 문제다
- 많은 사람은 과일을 싫어하지 않는다.
- 다만 손질, 보관, 실패 리스크, 가격 납득 문제 때문에 우선순위에서 밀린다.

### 가설 2. 사람들은 과일 자체보다 `상황 해결`에 돈을 낸다
- 아침을 간단히 해결하고 싶다.
- 아이 간식을 덜 죄책감 있게 준비하고 싶다.
- 부모 건강을 챙기고 싶다.
- 군것질 대신 덜 부담스럽게 먹고 싶다.

### 가설 3. 세그먼트마다 돈을 내는 명분이 다르다
- 어떤 집단은 `건강`
- 어떤 집단은 `편의`
- 어떤 집단은 `돌봄`
- 어떤 집단은 `안심`
- 어떤 집단은 `체중관리`

### 가설 4. 반복 구매보다 먼저 반복 섭취 루틴이 만들어져야 한다
- 한 번 사게 만드는 것만으로는 부족하다.
- `언제`, `어떻게`, `누가`, `왜` 먹는지가 설계되어야 재구매가 생긴다.

## 4. 세그먼트 설계 기준
- 먹는 시간대: 아침, 간식, 식후, 선물 후 공유, 돌봄 상황
- 소비 주체: 본인, 부모, 아이, 가족, 동료
- 지불 명분: 건강, 편의, 안심, 돌봄, 자기관리, 체면
- 마찰 유형: 가격, 손질, 보관, 품질 불안, 선택 피로, 루틴 부재
- 반복 가능성: 낮음 / 중간 / 높음

## 5. 우선 세그먼트 초안

### 5-1. 아침 대체형
- 상황:
  - 출근/등교 전 간단히 먹을 수 있는 것이 필요하다.
- 주요 마찰:
  - 손질 귀찮음
  - 보관 불편
  - 빵/음료 대비 약한 편의성
- 지불 명분:
  - 아침을 덜 무겁게 해결
  - 건강한 출발
- 환경 설계 방향:
  - 바로 먹는 소량 구성
  - 아침용 묶음
  - 냉장고에서 꺼내 바로 먹는 UX
- 우선도:
  - 중

### 5-2. 아이 간식 관리형
- 상황:
  - 부모가 과자 대신 덜 죄책감 있는 간식을 찾는다.
- 주요 마찰:
  - 아이가 안 먹을까 봄
  - 보관과 손질이 귀찮음
  - 가격 대비 실패 리스크 큼
- 지불 명분:
  - 아이 건강
  - 부모의 안심
  - 간식 죄책감 완화
- 환경 설계 방향:
  - 아이가 먹기 쉬운 소분
  - 실패 적은 품목 큐레이션
  - 보관/섭취 가이드
- 우선도:
  - 상

### 5-3. 가족 건강 루틴형
- 상황:
  - 장보기 안에서 가족용 건강 식품으로 반복 구매한다.
- 주요 마찰:
  - 늘 사지만 자주 버리게 됨
  - 대용량 부담
  - 구매 후 섭취 루틴 약함
- 지불 명분:
  - 가족 건강 관리
  - 믿고 사는 식탁 습관
- 환경 설계 방향:
  - 가족용 반복 SKU
  - 소비 속도에 맞춘 중간 용량
  - 재구매 주기 리마인드
- 우선도:
  - 상

### 5-4. 1인 자기관리형
- 상황:
  - 다이어트, 체중관리, 건강 관리를 위해 가볍게 먹고 싶다.
- 주요 마찰:
  - 과일도 당이 높다는 인식
  - 혼자 먹기엔 양이 많음
  - 습관으로 굳지 않음
- 지불 명분:
  - 간식 대체
  - 덜 부담스러운 자기관리
- 환경 설계 방향:
  - 소용량
  - 자기관리 문법
  - 저부담 섭취 메시지
- 우선도:
  - 중

### 5-5. 직장/오피스 간식형
- 상황:
  - 회사에서 커피, 빵, 과자 대신 가볍게 먹을 것을 찾는다.
- 주요 마찰:
  - 휴대와 공유 불편
  - 냄새/쓰레기/보관 문제
- 지불 명분:
  - 덜 죄책감 있는 간식
  - 팀 공유
- 환경 설계 방향:
  - 책상형/오피스형 패키지
  - 나눠먹기 쉬운 구성
- 우선도:
  - 실험

### 5-6. 돌봄/안부형
- 상황:
  - 부모 건강, 회복기 가족, 안부 챙김 상황에서 구매한다.
- 주요 마찰:
  - 뭘 보내야 할지 모름
  - 도착 상태와 만족 반응이 중요함
- 지불 명분:
  - 돌봄 표현
  - 실용적 안부
- 환경 설계 방향:
  - 선물보다 덜 부담스러운 돌봄 패키지
  - 받는 사람 상태별 추천
- 우선도:
  - 상

### 5-7. 선물-전환형
- 상황:
  - 선물 구매는 잘 일어나지만 일상 섭취로 이어지지 않는다.
- 주요 마찰:
  - 선물은 이벤트성으로 끝남
  - 재구매 루프 부재
- 지불 명분:
  - 체면과 안심
- 환경 설계 방향:
  - 선물 후 일상형 SKU 연결
  - 동일 취향 후속 제안
- 우선도:
  - 상

## 6. 세그먼트별 실행 프레임
각 세그먼트는 아래 형식으로 구체화한다.

| 항목 | 질문 |
|---|---|
| 상황 | 이 사람은 언제 과일을 먹으려 하는가 |
| 지불 명분 | 왜 이 상황에서 돈을 낼 이유가 생기는가 |
| 핵심 마찰 | 돈을 못 내게 만드는 가장 큰 장애는 무엇인가 |
| 환경 장치 | 무엇을 바꾸면 먹는 확률이 올라가는가 |
| 제품 구조 | 어떤 SKU/구성/용량이 필요한가 |
| 메시지 | 어떤 문장이 지불 결정을 돕는가 |
| 루틴 장치 | 재구매보다 먼저 무엇이 습관화를 만드는가 |
| 검증 | 어떤 실험으로 빠르게 확인할 것인가 |

## 7. 우선순위 제안

### 즉시 공략군
- 아이 간식 관리형
- 가족 건강 루틴형
- 돌봄/안부형
- 선물-전환형

이유:
- 지불 명분이 선명하다.
- 구매 주체가 명확하다.
- 안심, 건강, 돌봄 같은 강한 문법이 있다.
- 환경 설계 효과를 비교적 빠르게 검증할 수 있다.

### 후속 실험군
- 아침 대체형
- 1인 자기관리형
- 직장/오피스 간식형

이유:
- 시장성은 있지만 메시지와 포맷 검증이 더 필요하다.
- 대체재가 강해 차별 포인트를 더 정교하게 잡아야 한다.

## 8. 실행 단계

### Phase 1. 세그먼트 정리
- 기존 타겟 문서를 상황 중심 세그먼트로 재구성
- 세그먼트별 대표 상황 1문장 정의

### Phase 2. 마찰 지도 작성
- 각 세그먼트별 핵심 마찰 3개 이내로 압축
- `가격`, `편의`, `안심`, `루틴` 중 무엇이 가장 큰지 표시

### Phase 3. 환경 장치 설계
- 소포장
- 손질/보관 보조
- 섭취 타이밍 제안
- 실패 방지 장치
- 가족 공유 구조

### Phase 4. 오퍼 구조 설계
- 진입 상품
- 반복 상품
- 브리지 상품
- 선물 -> 일상 전환 상품

### Phase 5. 루틴 설계
- 첫 구매 후 첫 섭취 성공 장치
- 재섭취 리마인드
- 다음 구매 제안

### Phase 6. 실험 설계
- 세그먼트별 대표 랜딩 메시지
- 세그먼트별 대표 SKU
- 세그먼트별 성공 지표 설정

## 9. 세그먼트별 성공 지표 예시
- 아이 간식 관리형: 첫 구매 후 30일 내 재구매율
- 가족 건강 루틴형: 가정용 반복 SKU 전환율
- 돌봄/안부형: 선물 후 후속 일상형 전환율
- 선물-전환형: 선물 구매자 중 일상 SKU 진입 비율
- 1인 자기관리형: 소용량 SKU 체험 후 재구매율

## 10. 당장 만들 문서
- 세그먼트 맵
- 세그먼트별 마찰 지도
- 세그먼트별 환경 장치 표
- 세그먼트별 메시지/오퍼 표
- 우선 실험 3개 정의서

## 11. 바로 다음 액션
1. 기존 `타겟` 파트를 상황 중심 세그먼트 구조로 다시 쓴다.
2. 즉시 공략군 4개에 대해 마찰/명분/환경 장치를 표로 만든다.
3. 각 세그먼트별 대표 SKU 가설과 대표 메시지 1개씩 뽑는다.
4. 그 다음에야 상세페이지나 채널 전략으로 내려간다.
