# 온브릭스 벤치마크

## 1. 기준선 요약
- 온브릭스는 “프리미엄 과일 전문 브랜드” 중 현재 가장 명확한 비교 대상이다.
- 핵심 메시지는 `당도`, `선별`, `큐레이션`, `선물`, `전용 앱`, `브랜드 패키지`다.

## 2. 온브릭스 프로파일

| 항목 | 내용 |
|------|------|
| 브랜드명 | 온브릭스 |
| 포지션 | 프리미엄 과일 전문 브랜드 |
| 핵심 고객 | 선물 수요층, 품질 중시 고객, 프리미엄 과일 소비자 |
| 핵심 가치 제안 | 시중보다 높은 당도, 엄격한 선별, 브랜드화된 선물 경험 |
| 대표 경쟁 축 | 당도, 선물세트, 앱 기반 구매 경험, 체험형 팝업 |
| 핵심 채널 | 모바일 앱, 자체몰, 오프라인 팝업 |

## 3. 핵심 근거
1. App Store 설명에서 온브릭스는 `당도`, `크기`, `외형`의 3원칙을 전면에 내세운다. 시중보다 `1~2 brix` 높은 원물을 선별한다고 명시한다. 출처: [App Store 온브릭스](https://apps.apple.com/kr/app/%EC%98%A8%EB%B8%8C%EB%A6%AD%EC%8A%A4/id6445912643)
2. 2024년 1월 기준 누적 매출 `1,030억`, 연간 출고 `130만 건`, 2023년 매출 `약 418억`을 기록했다. 출처: [Platum](https://platum.kr/archives/221628)
3. 2025년 설 과일 선물세트는 선주문 `12만 개 이상`을 기록했다. 출처: [아주경제](https://www.ajunews.com/view/20250120110542803)
4. 2024년과 2025년 팝업스토어 기사에서 온브릭스는 “당도 측정 체험”을 오프라인 브랜딩 경험으로 활용했다. 출처: [아주경제 팝업](https://www.ajunews.com/view/20240807170059899), [아시아타임즈](https://www.asiatime.co.kr/article/20250807500110)

## 4. 온브릭스의 강점
1. 브랜드가 매우 단순하고 명확하다. “브릭스=당도”로 기억 포인트가 선명하다.
2. 품질 기준이 감성 문장이 아니라 수치 언어로 표현된다.
3. 선물세트와 패키지 경험이 브랜드 메시지와 잘 맞는다.
4. 앱, 멤버십, 쿠폰, 스토리 콘텐츠를 모두 가지고 있어 반복 구매 구조가 있다.
5. 팝업스토어를 통해 온라인 과일 브랜드의 약점인 체험 부재를 보완한다.

## 5. 온브릭스의 약점
1. 메시지의 중심이 `당도`에 치우쳐 있어, 식감/취향/상황/TPO 추천으로 확장 여지가 있다.
2. 프리미엄 중심 포지션이라 가격 저항이 있는 고객층으로 확장하기 어렵다.
3. 선물과 프리미엄 큐레이션에는 강하지만, B2B 도매/식당/소매사업자 문제 해결 포지션은 약하다.
4. 구매 이유가 “좋은 과일”에 집중돼 있어, ERP/발주/가격 변동/사업자 운영처럼 구조적 문제 해결 스토리는 약하다.

## 6. 우리가 이겨야 하는 축
| 축 | 온브릭스 강점 | 우리가 이기려면 필요한 것 |
|----|---------------|---------------------------|
| 품질 언어 | 당도 1~2brix 상향, 3원칙 선별 | 당도뿐 아니라 품종/식감/상황/TPO/재구매 데이터까지 확장 |
| 선물 경험 | 패키지와 선물세트 강함 | 선물뿐 아니라 일상 반복구매와 B2B까지 연결 |
| 브랜드 | 프리미엄 과일 전문 브랜드로 인식 | 더 좁고 더 강한 문제 정의가 필요 |
| 채널 | 앱과 팝업 운영 | 스마트스토어/네이버/오프라인 전환까지 포함한 멀티채널 설계 |
| 운영 해자 | 선별과 소싱 | B2B 공급망, 가격 이원화, 발주 자동화, 폐기 제어까지 포함한 운영형 해자 |

## 7. 1차 결론
- 온브릭스를 같은 언어로 이기는 전략은 어렵다.
- “더 비싸고 더 좋은 과일”만으로는 후발주자가 차별화하기 어렵다.
- 승부축은 아래 3가지 중 하나로 재정의해야 한다.
1. `프리미엄 + 결정 대행`: 당도 중심이 아니라 취향/상황 기반 추천으로 확장
2. `프리미엄 + B2B/B2C 연결`: 일반 소비자와 사업자 문제를 동시에 푸는 구조
3. `프리미엄 + 운영 데이터`: 가격 변동, 발주, 재구매, 폐기 통제까지 보이는 구조
